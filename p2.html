Project #2 Face Generator

Description

The project above was an experiment in parameterized design where I've tried to create multiple variations of the same form by using multiple shapes and setting constraints to those varieties. This project also question what defines certain figures, like a face. What are the constraints that make those shapes recognizable?

Design Process

From the beginning, I wanted to do something more abstract. So instead of starting by drawing precise versions of my face, I simply defined, broadly, how far (or how close) each component of my random face could go while still being considered a face.

From that point, I started to think about each component separately. "What sorts of shapes can be read as an eye?", "What can I draw noses?", "How can I simplify a mouth?". I kept questioning those things and then I decided between a couple of elements to randomize on each part of my randomized face.

"What is a face?"

Finally, I began to think in different ways to interact with my figure in order to randomize it. I choose to create some variations according to which part of the screen the user decided to click. For instance, the main shape changes according to the side of the screen. Also, the smile will vary according to height. From smiley to neutral to sad.

Reflection

It's easy to find yourself so focused on the parameters and numbers and conditions that you simply forget about what is that you're doing. What kind of exceptions your object needs? Are you including everything that is necessary to build that system?
